# 設計概要

> **対象読者**: 開発者、アーキテクト、コントリビューター

wasm-num の設計上の判断は、WebAssembly 仕様の形式化という目標によって駆動されています。本セクションでは、基盤となる設計哲学、採用しているパターン、主要なトレードオフ、そしてアーキテクチャ決定記録（ADR）を文書化しています。

## クイックサマリー

wasm-num は以下のコア原則に基づいて構築されています：

1. **仕様がソースオブトゥルース** — すべての定義は WebAssembly 仕様から直接派生
2. **BitVec N を統一表現として使用** — すべての Wasm 数値型は `BitVec N` のエイリアス
3. **Set α による非決定性のモデリング** — 仕様で許容される結果をすべて捕捉
4. **定義と証明の厳格な分離** — コンパイル時間の最適化と関心の分離
